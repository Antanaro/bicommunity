# Заполнение базы данных категориями и темами по BI

Этот скрипт заполняет базу данных стандартными категориями и темами, связанными с Business Intelligence (BI).

## Что будет добавлено

Скрипт создаст **11 категорий** по различным аспектам BI:

1. **Хранилища данных (Data Warehousing)** - проектирование и управление DWH
2. **ETL процессы** - извлечение, трансформация и загрузка данных
3. **Отчетность и дашборды** - создание отчетов и аналитических панелей
4. **Аналитика и BI инструменты** - Power BI, Tableau, Qlik и другие платформы
5. **Визуализация данных** - принципы и практики визуализации
6. **OLAP и многомерный анализ** - OLAP кубы, MDX запросы
7. **Data Mining и машинное обучение** - применение ML в BI
8. **Управление данными (Data Governance)** - политики, качество данных, метаданные
9. **Интеграция данных** - интеграция из различных источников
10. **Производительность и оптимизация** - оптимизация BI систем
11. **Общие вопросы BI** - карьера, обучение, тренды

В каждой категории будет создано по **2 примерные темы** для начала обсуждений.

## Как использовать

### Вариант 1: Через bat-файл (Windows)

Просто запустите:
```bash
seed-bi-data.bat
```

### Вариант 2: Через npm

```bash
npm run seed-bi
```

### Вариант 3: Напрямую через Node.js

```bash
node src/scripts/seed-bi-data.js
```

## Требования

- База данных должна быть создана и настроена
- Миграции должны быть выполнены (таблицы `categories`, `topics`, `users` должны существовать)
- Файл `.env` должен быть настроен с правильными параметрами подключения к БД

## Повторный запуск

Скрипт безопасен для повторного запуска:
- Если категория уже существует, она будет пропущена
- Если тема уже существует, она не будет создана повторно
- Скрипт не удаляет существующие данные

## Примечания

- Если в базе нет пользователей, скрипт создаст системного пользователя `system` для создания тем
- Все созданные темы будут привязаны к первому найденному пользователю (или системному)
- Вы можете вручную редактировать категории и темы после создания через админ-панель или API
